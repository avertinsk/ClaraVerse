# Ollama (локальный)

Запускайте ИИ-модели локально на вашем собственном оборудовании с Ollama. Без API-ключа, без зависимости от облака, полностью бесплатно и конфиденциально.

## Установка Ollama

1. Перейдите на [ollama.com](https://ollama.com/) и загрузите установщик для вашей ОС (macOS, Linux или Windows).
2. Запустите установщик и следуйте инструкциям.
3. Проверьте установку:

```bash
ollama --version
```

Ollama запускает локальный API-сервер на порту `11434` по умолчанию.

## Загрузка модели

Загрузите хотя бы одну модель перед подключением к ClaraVerse:

```bash
# Общего назначения
ollama pull llama3.1

# Меньше / быстрее
ollama pull llama3.1:8b

# Для кода
ollama pull codellama

# Лёгкий чат
ollama pull phi3
```

Просмотрите все доступные модели на [ollama.com/library](https://ollama.com/library).

## Подключение Ollama к ClaraVerse

1. Откройте **Панель администратора > Провайдеры**.
2. Нажмите **Добавить провайдер**.
3. Введите имя (например, «Ollama»).
4. Установите базовый URL `http://localhost:11434/v1` (или IP-адрес хоста Ollama, если работает на другой машине).
5. Оставьте поле API-ключа пустым — Ollama не требует его.
6. Нажмите **Сохранить**.

Все загруженные вами модели автоматически появятся в селекторах моделей.

## Рекомендуемые модели

| Модель | Размер | Лучше всего для |
|-------|------|----------|
| **llama3.1:70b** | 40 ГБ | Высококачественных общих задач (нужен мощный GPU) |
| **llama3.1:8b** | 4.7 ГБ | Хороший универсал для большинства硬件 |
| **codellama:13b** | 7.4 ГБ | Генерации кода и отладки |
| **phi3:mini** | 2.3 ГБ | Быстрых ответов на ограниченном оборудовании |
| **mistral:7b** | 4.1 ГБ | Сбалансированной производительности и скорости |
| **gemma2:9b** | 5.4 ГБ | Сильных рассуждений при малом размере |

Выбирайте в зависимости от доступной VRAM. Как правило, вам нужно примерно 1 ГБ VRAM на 1-2 миллиарда параметров (квантованных).

## Советы

- **Полностью офлайн.** После загрузки модель работает без подключения к интернету. Идеально для изолированных сред.
- **Ускорение GPU.** Ollama автоматически использует ваш GPU, если доступен (NVIDIA, AMD или Apple Silicon). Проверьте `ollama ps`, чтобы убедиться, что модель загружена на GPU.
- **Несколько моделей.** Вы можете загрузить столько моделей, сколько вмещает ваш диск. Только активно загруженные модели потребляют VRAM.
- **Удалённый Ollama.** Если Ollama работает на другой машине (например, GPU-сервер), установите `OLLAMA_HOST=0.0.0.0` на этой машине и направьте ClaraVerse на `http://<server-ip>:11434`.
- **Обновляйте модели.** Периодически запускайте `ollama pull <model>` для получения последней версии модели.
- **Сеть Docker.** Если ClaraVerse и Ollama оба работают в Docker, используйте IP хоста Docker или сетевой псевдоним вместо `localhost`. На Docker Desktop обычно работает `host.docker.internal`.
