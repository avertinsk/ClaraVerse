# Настройка OpenAI

Подключите ClaraVerse к OpenAI для использования GPT-4o, o1, o3 и других моделей OpenAI.

## Получение API-ключа

1. Перейдите на [platform.openai.com](https://platform.openai.com/) и войдите (или создайте аккаунт).
2. Перейдите в **API keys** на левой боковой панели (или посетите [platform.openai.com/api-keys](https://platform.openai.com/api-keys)).
3. Нажмите **Create new secret key**, дайте имя и скопируйте ключ. Вы не сможете увидеть его снова.
4. Убедитесь, что биллинг включён в **Settings > Billing** — доступ к API требует платного аккаунта.

## Добавление OpenAI в ClaraVerse

1. Откройте **Панель администратора > Провайдеры**.
2. Нажмите **Добавить провайдер**.
3. Введите имя (например, «OpenAI»).
4. Установите базовый URL `https://api.openai.com/v1`.
5. Вставьте ваш API-ключ.
6. Нажмите **Сохранить**.

Модели будут получены автоматически и появятся в селекторах моделей по всей платформе.

## Доступные модели

| Модель | Лучше всего для |
|-------|----------|
| **GPT-4o** | Общего назначения, мультимодальность (текст + зрение) |
| **GPT-4o mini** | Быстрых, экономичных задач |
| **o1** | Сложных рассуждений, математики, кода |
| **o3** | Продвинутых рассуждений с расширенным мышлением |
| **o3-mini** | Лёгких рассуждений |
| **GPT-4.1** | Задач с длинным контекстом (окно 1M токенов) |
| **GPT-4.1 mini** | Экономичных задач с длинным контекстом |

Доступность моделей зависит от уровня вашего аккаунта OpenAI.

## Советы

- **Установите лимиты расходов.** Используйте страницу лимитов использования OpenAI, чтобы ограничить ежемесячные расходы и избежать сюрпризов.
- **Выбирайте правильную модель.** GPT-4o mini хорошо справляется с большинством повседневных задач и стоит значительно меньше, чем GPT-4o. Оставьте o1/o3 для задач, требующих глубоких рассуждений.
- **Ключи организации.** Если у вас есть организация OpenAI, убедитесь, что используете API-ключ, связанный с этой организацией, чтобы расходы были правильно учтены.
- **Лимиты скорости.** OpenAI применяет ограничения токенов и запросов в минуту на основе вашего уровня использования. Если пользователи сталкиваются с ошибками скорости, рассмотрите обновление уровня или распределение нагрузки между несколькими ключами.
