# Пользовательские OpenAI-совместимые провайдеры

Любой API-сервер, реализующий формат завершения чата OpenAI (`/v1/chat/completions`), работает с ClaraVerse. Это охватывает широкий спектр локальных движков инференса, облачных платформ и прокси-сервисов.

## Добавление пользовательского провайдера

1. Откройте **Панель администратора > Провайдеры**.
2. Нажмите **Добавить провайдер**.
3. Введите имя (например, «LM Studio», «Together AI»).
4. Установите базовый URL на API-endpoint (например, `http://localhost:1234/v1`).
5. Введите API-ключ, если провайдер требует его, или оставьте пустым для локальных движков.
6. Нажмите **Сохранить**.

ClaraVerse попытается получить список моделей из endpoint `/v1/models`. Если это сработает, модели появятся автоматически. Если нет, вы можете добавить ID моделей вручную.

## Совместимые сервисы

### Локальный инференс

| Сервис | Типичный базовый URL | API-ключ | Примечания |
|---------|-----------------|---------|-------|
| **LM Studio** | `http://localhost:1234/v1` | Нет | GUI-based, удобное управление моделями |
| **text-generation-webui** (oobabooga) | `http://localhost:5000/v1` | Нет | Поддерживает множество форматов моделей |
| **vLLM** | `http://localhost:8000/v1` | Нет | Высокопроизводительное обслуживание, продакшен-уровень |
| **llama.cpp server** | `http://localhost:8080/v1` | Нет | Минимальный footprint |
| **LocalAI** | `http://localhost:8080/v1` | Нет | Прямая замена OpenAI |

### Облачные платформы

| Сервис | Базовый URL | API-ключ | Примечания |
|---------|----------|---------|-------|
| **Together AI** | `https://api.together.xyz/v1` | Да | Широкий выбор моделей, быстрый инференс |
| **Groq** | `https://api.groq.com/openai/v1` | Да | Чрезвычайно быстрый инференс (аппаратное обеспечение LPU) |
| **Fireworks AI** | `https://api.fireworks.ai/inference/v1` | Да | Быстрый, экономичный |
| **Perplexity** | `https://api.perplexity.ai` | Да | Модели с поисковым дополнением |
| **Mistral AI** | `https://api.mistral.ai/v1` | Да | Модели Mistral и Mixtral |
| **DeepSeek** | `https://api.deepseek.com/v1` | Да | DeepSeek-R1, DeepSeek-V3 |
| **OpenRouter** | `https://openrouter.ai/api/v1` | Да | Маршрутизация к 100+ моделям из одного ключа |

## Пример настройки: LM Studio

1. Откройте LM Studio и загрузите модель из браузера моделей.
2. Перейдите на вкладку **Local Server** и нажмите **Start Server**. Порт по умолчанию `1234`.
3. В ClaraVerse добавьте пользовательский провайдер с базовым URL `http://localhost:1234/v1` и без API-ключа.
4. Загруженная модель появится в селекторах моделей ClaraVerse.

## Пример настройки: Together AI

1. Зарегистрируйтесь на [together.ai](https://www.together.ai/) и скопируйте ваш API-ключ с панели управления.
2. В ClaraVerse добавьте пользовательский провайдер:
   - **Базовый URL:** `https://api.together.xyz/v1`
   - **API-ключ:** ваш ключ Together AI
3. Станут доступны модели, такие как Llama 3.1 405B, Mixtral и Code Llama.

## Советы

- **Сначала протестируйте endpoint.** Выполните быстрый curl, чтобы убедиться, что API доступен, перед добавлением в ClaraVerse:
  ```bash
  curl http://localhost:1234/v1/models
  ```
- **Сеть Docker.** Если ClaraVerse и сервер инференса оба работают в Docker, используйте имя контейнера или `host.docker.internal` вместо `localhost`.
- **ID моделей.** Некоторые провайдеры используют разные форматы ID моделей. Если модели не заполняются автоматически, проверьте документацию провайдера для точной строки ID модели и добавьте её вручную.
- **Потоковая передача.** ClaraVerse использует потоковую передачу по умолчанию. Большинство OpenAI-совместимых API поддерживают это, но если вы видите проблемы, проверьте, требует ли ваш провайдер специфический параметр потоковой передачи.
- **Несколько провайдеров.** Вы можете добавить один и тот же сервис несколько раз с разными конфигурациями — например, одна запись для локального экземпляра LM Studio и другая для Together AI. Каждый получает своё имя и список моделей.
