# Визуальный конструктор рабочих процессов

![Рабочие процессы](/img/features/workflows-screenshot.png)

Создавайте автоматизации, соединяя узлы на холсте. Перетаскивайте, размещайте, соединяйте — без написания кода. Рабочие процессы поддерживают параллельное выполнение, планирование и 150+ интеграций.

## Начало работы

1. Откройте **Агенты** на боковой панели (или карточку Рабочие процессы на панели управления).
2. Нажмите кнопку **+**, чтобы создать новый рабочий процесс агента.
3. Перетащите узлы из панели на холст.
4. Соедините выходы со входами, проведя рёбра между узлами.
5. Нажмите **Запуск** для выполнения.

## Генерация с помощью ИИ

Опишите, что вы хотите, обычным текстом, и позвольте LLM построить рабочий процесс за вас. Например:

> «Каждое утро получай открытые задачи GitHub с меткой 'bug', суммируй их и отправляй сводку в Slack #engineering.»

Clara генерирует полный граф узлов. Проверьте его, при необходимости отредактируйте и активируйте.

## Ключевые возможности

- **Параллельное выполнение.** Ветки выполняются одновременно. Рабочий процесс, разветвляющийся на пять API, завершается так же быстро, как самая медленная ветка, а не суммарно по всем пяти.
- **Планирование.** Установите расписание cron для рабочих процессов — ежечасно, ежедневно, еженедельно или произвольно.
- **API-триггеры.** Предоставьте рабочий процесс как HTTP-endpoint для вызова из внешних приложений или вебхуков.
- **150+ интеграций.** Используйте те же интеграции, что доступны в Чате и Nexus — Slack, GitHub, Google Sheets, базы данных и многое другое.
- **История версий.** Каждое сохранение создаёт версию. Откатитесь к любому предыдущему состоянию.
- **Условная логика.** Ветвление на основе значений данных, ответов API или решений ИИ.

## Типы узлов

| Тип | Что делает |
|---|---|
| **Триггер** | Запускает рабочий процесс (вручную, по расписанию, вебхук или событие) |
| **ИИ / LLM** | Отправляет промпт модели и возвращает ответ |
| **Интеграция** | Вызывает внешний сервис (Slack, GitHub и т.д.) |
| **Трансформация** | Изменяет данные — map, filter, merge, split |
| **Условие** | Направляет выполнение на основе проверки true/false |
| **Код** | Запускает пользовательский код для всего, что не покрывают встроенные узлы |

## Советы
