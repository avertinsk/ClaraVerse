# Локальный ИИ

Запускайте ИИ-модели локально на вашем собственном оборудовании с Ollama или LM Studio. Без API-ключей, без зависимости от облака, полностью бесплатно и конфиденциально. Вы также можете комбинировать локальные и облачные провайдеры для гибридной настройки.

## Поддерживаемые локальные провайдеры

| Провайдер | Тип | Как подключить |
|----------|------|---------------|
| **Ollama** | Локальный инференс | Добавить как провайдер с базовым URL `http://localhost:11434/v1` |
| **LM Studio** | Локальный инференс (GUI) | Добавить как провайдер с базовым URL `http://localhost:1234/v1` |
| **Любой OpenAI-совместимый сервер** | Различный | Добавить как провайдер с базовым URL сервера |

## Настройка Ollama

1. Установите Ollama с [ollama.com](https://ollama.com/).
2. Загрузите модель: `ollama pull llama3.1`
3. В ClaraVerse перейдите в **Панель администратора > Провайдеры**, нажмите **Добавить провайдер**.
4. Введите имя «Ollama», базовый URL `http://localhost:11434/v1`, оставьте API-ключ пустым.
5. Сохраните. Ваши загруженные модели появятся в селекторах моделей.

**Если ClaraVerse работает в Docker**, Ollama на `localhost` не будет доступен изнутри контейнера. Исправьте это, настроив Ollama слушать все интерфейсы:

```bash
# Linux (systemd):
sudo systemctl edit ollama
```

Добавьте под `[Service]`:

```ini
Environment="OLLAMA_HOST=0.0.0.0"
```

Затем перезапустите:

```bash
sudo systemctl restart ollama
```

Используйте `http://host.docker.internal:11434/v1` (Docker Desktop) или IP-адрес вашей машины как базовый URL в ClaraVerse.

На macOS установите переменную окружения перед запуском:

```bash
OLLAMA_HOST=0.0.0.0 ollama serve
```

## Настройка LM Studio

1. Откройте LM Studio и загрузите нужные модели.
2. Запустите локальный сервер из интерфейса LM Studio (по умолчанию работает на порту 1234).
3. В ClaraVerse добавьте провайдер с именем «LM Studio», базовый URL `http://localhost:1234/v1`, без API-ключа.
4. Загруженная модель появится в селекторах моделей ClaraVerse.

## BYOK — Принесите свои ключи

ClaraVerse поддерживает гибридный подход:

- **Бесплатные локальные модели** — Используйте модели Ollama или LM Studio без затрат.
- **Облачные провайдеры** — Добавьте свои API-ключи для OpenAI, Anthropic, Google или любого OpenAI-совместимого endpoint.
- **Комбинируйте** — Используйте локальную модель для повседневных задач и облачную модель для специализированной работы.

Вы никогда не привязаны к одному провайдеру. Переключайте модели для каждой беседы или задачи.

## Полностью офлайн-работа

Чтобы запустить ClaraVerse без зависимости от интернета:

1. Установите ClaraVerse через Docker (нужен интернет для начальной загрузки).
2. Установите Ollama и загрузите нужные модели, пока есть интернет.
3. Установите `OLLAMA_HOST=0.0.0.0`, чтобы ClaraVerse могReach Ollama из Docker.
4. Отключитесь от интернета. ClaraVerse и Ollama работают полностью локально.

Все беседы хранятся в вашей локальной базе данных. Ничего не отправляется вовне.

## Советы

- Локальные модели работают на вашем оборудовании. Убедитесь, что у вас достаточно ОЗУ и (опционально) GPU. 8 ГБ ОЗУ — минимум; 16 ГБ или более рекомендуется для больших моделей.
- Ускорение GPU автоматическое с Ollama, если у вас есть поддерживаемый GPU (NVIDIA, AMD или Apple Silicon). Запустите `ollama ps`, чтобы убедиться, что модель загружена на GPU.
- Вы можете загрузить столько моделей Ollama, сколько вмещает ваш диск. Только активно загруженные модели потребляют VRAM.
- Если вы загрузили новые модели в Ollama, обновите провайдер в ClaraVerse, чтобы их подхватить.
