# Аналитика

Панель аналитики даёт администраторам видимость использования платформы. Доступ через **Панель администратора > Аналитика**.

## Обзор использования

Высокоуровневая статистика по всему экземпляру:

```http
GET /api/admin/analytics/overview
Authorization: Bearer <access_token>
```

| Метрика | Описание |
|--------|-------------|
| `total_users` | Все зарегистрированные пользователи |
| `active_chats` | Текущие активные чаты |
| `total_messages` | Сообщения за всё время на платформе |
| `api_calls_today` | API-вызовы за текущий день |
| `agent_executions` | Выполнения агентов за всё время |
| `agents_run_today` | Агенты, запущенные за текущий день |

## Аналитика провайдеров

Посмотрите, как используется каждый ИИ-провайдер:

```http
GET /api/admin/analytics/providers
Authorization: Bearer <access_token>
```

Метрики для каждого провайдера включают:

- **Всего запросов** — API-вызовы к этому провайдеру за всё время
- **Запросов сегодня** — Вызовы за текущий день
- **Использовано токенов** — Общее потребление токенов
- **Среднее время отклика** — Средняя задержка в миллисекундах

:::tip
Используйте аналитику провайдеров для выявления热点 затрат. Если один провайдер доминирует в потреблении токенов, рассмотрите добавление более дешёвой альтернативы для повседневных задач.
:::

## Аналитика чатов

Агрегированные данные о беседах по всей платформе:

```http
GET /api/admin/analytics/chats
Authorization: Bearer <access_token>
```

Отслеживает общее количество чатов, сообщений и паттернов бесед с течением времени.

## Статистика использования моделей

Разбивка по отдельным моделям:

```http
GET /api/admin/analytics/models
Authorization: Bearer <access_token>
```

| Метрика | Описание |
|--------|-------------|
| `total_requests` | Запросы к этой модели за всё время |
| `requests_today` | Запросы за текущий день |
| `avg_tokens_per_request` | Среднее количество токенов на вызов |
| `success_rate` | Процент успешных ответов |

Модели с низким процентом успешных ответов могут указывать на проблемы конфигурации или проблемы на стороне провайдера.

## Метрики выполнения агентов

Отслеживайте активность агентов Nexus:

```http
GET /api/admin/analytics/agents
Authorization: Bearer <access_token>
```

Охватывает общее количество созданных агентов, количество выполнений и тенденции. Полезно для понимания, насколько активно используется система агентов и какие агенты создают наибольший трафик.

## Фоновые задачи

ClaraVerse запускает запланированные задачи обслуживания, которые отображаются в логах backend:

| Задача | Расписание | Назначение |
|-----|----------|---------|
| Очистка хранения | Ежедневно в 2:00 UTC | Удаляет просроченные данные согласно политике хранения уровня |
| Проверка льготного периода | Ежечасно | Обрабатывает понижение уровня |
| Истечение промо | Ежечасно | Истекают промо-обновления уровня |

Мониторьте их в логах контейнера backend:

```bash
docker logs claraverse-backend -f
```
