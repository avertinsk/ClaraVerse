# Управление пользователями

Управляйте каждым пользователем на вашем экземпляре ClaraVerse через **Панель администратора > Пользователи**.

## Просмотр пользователей

Список пользователей поддерживает поиск и фильтрацию:

| Фильтр | Опции |
|--------|---------|
| **Поиск** | По ID пользователя или домену |
| **Уровень** | `free`, `pro`, `max`, `enterprise` |

Результаты разбиты на страницы (по умолчанию 50 на страницу).

```http
GET /api/admin/users?page=1&page_size=50&search=john&tier=pro
Authorization: Bearer <access_token>
```

Каждая запись пользователя показывает ID пользователя (анонимизированный), Домен, Уровень, Чаты, Сообщения, Агенты, Последняя активность и Создан. Данные соответствуют GDPR и анонимизированы — полные адреса электронной почты не отображаются.

## Детали пользователя

- **Лимиты** — Значения по умолчанию на основе уровня (сообщений в день, выполнений в день, дни хранения)
- **Переопределения** — Любые корректировки, применённые администратором к этим лимитам
- **Использование** — Текущее потребление (сообщений сегодня, выполнений сегодня)

```http
GET /api/admin/users/:userID
Authorization: Bearer <access_token>
```

## Установка лимитов использования

Переопределите значения по умолчанию уровня для отдельных пользователей. Это полезно для продвинутых пользователей, которым нужны более высокие лимиты, или аккаунтов, которые следует ограничить.

```http
POST /api/admin/users/:userID/overrides
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "limits": {
    "messages_per_day": 2000,
    "executions_per_day": 200
  },
  "reason": "Продвинутому пользователю нужны более высокие лимиты"
}
```

Вы также можете переопределить уровень пользователя напрямую:

```http
POST /api/admin/users/:userID/overrides
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "tier": "pro",
  "reason": "Обновлено на испытательный период"
}
```

Чтобы удалить все переопределения и вернуть значения по умолчанию уровня:

```http
DELETE /api/admin/users/:userID/overrides
Authorization: Bearer <access_token>
```

:::tip
Документируйте причину любого переопределения. Увеличенные лимиты могут повлиять на потребление ресурсов и затраты на API.
:::

## Роли пользователей

ClaraVerse имеет две роли:

| Роль | Доступ |
|------|--------|
| `user` | Стандартный доступ к платформе |
| `admin` | Полный доступ к платформе плюс Панель администратора |

### Повышение пользователя до администратора

Установите роль напрямую в MongoDB:

```javascript
db.users.updateOne(
  { email: "user@example.com" },
  { $set: { role: "admin" } }
)
```

### SUPERADMIN_USER_IDS

Для дополнительного доступа администратора без изменения базы данных добавьте ID пользователей в переменную окружения `SUPERADMIN_USER_IDS` в вашем файле `.env`:

```bash
SUPERADMIN_USER_IDS=user-id-1,user-id-2,user-id-3
```

:::warning
Держите список ID суперадминистраторов коротким. Каждая учётная запись администратора — это точка привилегированного доступа.
:::
