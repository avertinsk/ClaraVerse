# Обновление ClaraVerse

ClaraVerse обновляется путём получения последнего кода и пересборки Docker-контейнеров.

## Стандартное обновление

```bash
# 1. Получите последние изменения
git pull

# 2. Остановите работающий стек
docker compose down

# 3. Пересоберите и запустите
docker compose up --build -d
```

Пересборка учитывает изменения кода как во фронтенде, так и в backend. Тома базы данных сохраняются автоматически — ваши беседы, пользователи и настройки переносятся.

## Резервное копирование перед обновлением

Хорошей практикой является создание резервной копии данных перед крупными обновлениями.

### Резервное копирование файла окружения

```bash
cp .env .env.backup
```

### Резервное копирование томов Docker

```bash
# Найдите имена ваших томов
docker volume ls | grep claraverse

# Резервное копирование данных MongoDB
docker run --rm -v claraverse_mongodb-data:/data -v $(pwd)/backups:/backup alpine \
  tar czf /backup/mongodb-backup-$(date +%Y%m%d).tar.gz -C /data .

# Резервное копирование данных MySQL
docker run --rm -v claraverse_mysql-data-new:/data -v $(pwd)/backups:/backup alpine \
  tar czf /backup/mysql-backup-$(date +%Y%m%d).tar.gz -C /data .
```

### Резервное копирование ключа шифрования

Ваш `ENCRYPTION_MASTER_KEY` в `.env` имеет критическое значение. Если вы его потеряете, все зашифрованные данные (беседы, сохранённые учётные данные) станут невосстановимыми. Сохраните копию в безопасном месте вне сервера.

## Обработкаbreaking changes

Иногда обновление может включать миграции базы данных или новые обязательные переменные окружения. Проверьте примечания к выпуску или журнал изменений перед обновлением. Если добавлены новые переменные `.env`:

```bash
# Сравните ваш .env с последним шаблоном
diff .env .env.example
```

Добавьте все отсутствующие переменные в ваш файл `.env` перед перезапуском.

## Полный сброс (ядерный вариант)

Если что-то пошло не так и вы хотите начать полностью заново:

```bash
# Остановить всё и удалить все тома (ЭТО УДАЛЯЕТ ВСЕ ДАННЫЕ)
docker compose down -v

# Пересобрать с нуля
docker compose up --build -d
```

:::danger
`docker compose down -v` удаляет все тома базы данных. Вы потеряете всех пользователей, беседы, настройки и загруженные файлы. Делайте это только если у вас есть резервные копии или вы готовы потерять всё.
:::
