# Конфигурация

Вся конфигурация выполняется через файл `.env` в корне проекта. Docker Compose автоматически читает этот файл. После изменения любого значения перезапустите стек:

```bash
docker compose down
docker compose up --build -d
```

## Режим окружения

| Переменная | По умолчанию | Описание |
|----------|---------|-------------|
| `ENVIRONMENT` | `development` | Установите `production` для продакшен-развёртываний. Отключает функции, доступные только в режиме разработки, например `DEV_API_KEY`. |

## URL-адреса фронтенда

Эти значения встраиваются во фронтенд на этапе сборки. Если вы их измените, необходимо пересобрать контейнер фронтенда.

| Переменная | По умолчанию | Описание |
|----------|---------|-------------|
| `VITE_API_BASE_URL` | `http://localhost:3001` | URL-адрес backend API, который вызывает фронтенд. Для продакшена: `https://api.yourdomain.com` |
| `VITE_WS_URL` | `ws://localhost:3001` | URL-адрес WebSocket для потоковой передачи в реальном времени. Для продакшена: `wss://api.yourdomain.com` |

## JWT-аутентификация

| Переменная | По умолчанию | Описание |
|----------|---------|-------------|
| `JWT_SECRET` | *(обязательно установить)* | Секретный ключ для подписи JWT-токенов. Минимум 32 символа. Сгенерируйте командой `openssl rand -hex 32`. |
| `JWT_ACCESS_TOKEN_EXPIRY` | `15m` | Срок действия токенов доступа. |
| `JWT_REFRESH_TOKEN_EXPIRY` | `168h` | Срок действия токенов обновления (по умолчанию: 7 дней). |

## Backend

| Переменная | По умолчанию | Описание |
|----------|---------|-------------|
| `ALLOWED_ORIGINS` | `http://localhost,http://localhost:5173,http://localhost:5174,http://localhost:3000,http://localhost:8080` | CORS-источники, разделённые запятыми. Для продакшена: `https://yourdomain.com` |
| `ENCRYPTION_MASTER_KEY` | *(обязательно установить)* | Шифрует пользовательские данные в MongoDB (беседы, учётные данные). Сгенерируйте командой `openssl rand -hex 32`. **Не теряйте этот ключ.** |
| `FRONTEND_URL` | `http://localhost:5173` | Используется backend для перенаправлений и ссылок в электронной почте. |
| `BACKEND_URL` | `http://localhost:3001` | Публичный URL-адрес backend, используется для генерации ссылок на скачивание. |

## База данных MySQL

| Переменная | По умолчанию | Описание |
|----------|---------|-------------|
| `MYSQL_ROOT_PASSWORD` | `claraverse_root_2024` | Пароль root для MySQL. Измените для продакшена. |
| `MYSQL_PASSWORD` | `claraverse_pass_2024` | Пароль для учётной записи MySQL `claraverse_user`. Измените для продакшена. |

Имя базы данных MySQL (`claraverse`) и пользователь (`claraverse_user`) задаются в `docker-compose.yml`.

## MongoDB и Redis

| Переменная | По умолчанию | Описание |
|----------|---------|-------------|
| `MONGODB_URI` | `mongodb://mongodb:27017/claraverse` | Строка подключения к MongoDB. Для Atlas: `mongodb+srv://user:pass@cluster.mongodb.net/claraverse` |
| `REDIS_URL` | `redis://redis:6379` | Строка подключения к Redis. Используется для планирования задач и WebSocket pub/sub. |

## Администрирование

| Переменная | По умолчанию | Описание |
|----------|---------|-------------|
| `SUPERADMIN_USER_IDS` | *(пусто)* | ID пользователей через запятую для предоставления прав суперадминистратора. Первый зарегистрированный пользователь автоматически становится администратором; используйте это для дополнительных администраторов. |

## Ограничение скорости

Все значения — запросы в минуту.

| Переменная | По умолчанию | Описание |
|----------|---------|-------------|
| `RATE_LIMIT_GLOBAL_API` | `200` | Глобальный лимит для всех маршрутов `/api/*`. |
| `RATE_LIMIT_PUBLIC_READ` | `120` | Лимит для публичных конечных точек только для чтения. |
| `RATE_LIMIT_AUTHENTICATED` | `60` | Лимит для запросов аутентифицированных пользователей. |
| `RATE_LIMIT_WEBSOCKET` | `20` | Лимит для новых WebSocket-подключений. |
| `RATE_LIMIT_IMAGE_PROXY` | `60` | Лимит для запросов прокси изображений (предотвращает злоупотребление пропускной способностью). |

## Контрольный список для продакшена

При развёртывании в продакшене как минимум:

1. Установите `ENVIRONMENT=production`.
2. Сгенерируйте надёжные уникальные значения для `JWT_SECRET` и `ENCRYPTION_MASTER_KEY`.
3. Измените `MYSQL_ROOT_PASSWORD` и `MYSQL_PASSWORD` от значений по умолчанию.
4. Установите `VITE_API_BASE_URL` и `VITE_WS_URL` на ваш публичный домен (с `https://` и `wss://`).
5. Установите `ALLOWED_ORIGINS` только на ваш домен фронтенда.
6. Установите `FRONTEND_URL` и `BACKEND_URL` на ваши публичные URL-адреса.
7. Разместите обратный прокси (nginx, Caddy, Traefik) перед сервисами для обработки TLS.
