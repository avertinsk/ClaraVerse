# Справочник переменных окружения

ClaraVerse использует единый файл `.env` в корне проекта как источник истины для всей конфигурации. Docker Compose читает его автоматически.

```bash
cp .env.example .env
```

## Режим окружения

| Переменная | По умолчанию | Обязательна | Описание |
|----------|---------|----------|-------------|
| `ENVIRONMENT` | `development` | Нет | Установите `production` для продакшен-развёртываний. Отключает функции только для разработки, такие как `DEV_API_KEY`. |

## Безопасность (обязательные)

Эти переменные **должны** быть установлены перед первым запуском. Сгенерируйте их с помощью `openssl`:

```bash
echo "ENCRYPTION_MASTER_KEY=$(openssl rand -hex 32)" >> .env
echo "JWT_SECRET=$(openssl rand -hex 64)" >> .env
```

| Переменная | По умолчанию | Обязательна | Описание |
|----------|---------|----------|-------------|
| `ENCRYPTION_MASTER_KEY` | *(пусто)* | **Да** | 256-битный hex-ключ для AES-256-GCM шифрования пользовательских данных (учётные данные, синхронизация чата). **Потеря этого ключа означает потерю доступа ко всем зашифрованным данным.** |
| `JWT_SECRET` | `change_me_to_a_random_secret_at_least_32_chars` | **Да** | Секрет для подписи JWT access и refresh токенов. Минимум 32 символа. |

## JWT-аутентификация

| Переменная | По умолчанию | Обязательна | Описание |
|----------|---------|----------|-------------|
| `JWT_ACCESS_TOKEN_EXPIRY` | `15m` | Нет | Время жизни access токена. Формат длительности Go (например, `15m`, `1h`). |
| `JWT_REFRESH_TOKEN_EXPIRY` | `168h` | Нет | Время жизни refresh токена. По умолчанию 7 дней. |

## Аргументы сборки фронтенда

Эти значения встраиваются во фронтенд при Docker-сборке. Если вы их измените, пересоберите контейнер фронтенда.

| Переменная | По умолчанию | Обязательна | Описание |
|----------|---------|----------|-------------|
| `VITE_API_BASE_URL` | `http://localhost:3001` | Нет | URL-адрес backend API, видимый браузеру. Для продакшена: `https://api.yourdomain.com`. |
| `VITE_WS_URL` | `ws://localhost:3001` | Нет | URL-адрес WebSocket, видимый браузеру. Для продакшена: `wss://api.yourdomain.com`. |

## Конфигурация backend

| Переменная | По умолчанию | Обязательна | Описание |
|----------|---------|----------|-------------|
| `ALLOWED_ORIGINS` | `http://localhost,http://localhost:5173,http://localhost:5174,http://localhost:3000,http://localhost:8080` | Нет | Разделённые запятыми разрешённые CORS-источники. Для продакшена установите только ваш домен. |
| `FRONTEND_URL` | `http://localhost:5173` | Нет | URL-адрес фронтенда, используемый backend для перенаправлений и ссылок. |
| `BACKEND_URL` | `http://localhost:3001` | Нет | Публичный URL-адрес backend, используемый для генерации абсолютных URL-адресов скачивания. |
| `DEV_API_KEY` | `claraverse-dev-key-2024` | Нет | API-ключ только для разработки для тестирования. Работает только когда `ENVIRONMENT` не `production`. **Примечание:** Это внутренняя переменная Docker, устанавливаемая автоматически `docker-compose.yml`. Вам не нужно добавлять её в ваш файл `.env`. |

## База данных MySQL

| Переменная | По умолчанию | Обязательна | Описание |
|----------|---------|----------|-------------|
| `MYSQL_ROOT_PASSWORD` | `claraverse_root_2024` | Нет | Пароль root MySQL. Измените для продакшена. |
| `MYSQL_PASSWORD` | `claraverse_pass_2024` | Нет | Пароль для учётной записи MySQL `claraverse_user`. Измените для продакшена. |
| `DATABASE_URL` | *(автогенерируется из `MYSQL_PASSWORD`)* | Нет | Полная строка подключения к MySQL. Переопределите для использования внешнего экземпляра MySQL. **Примечание:** Это внутренняя переменная Docker, устанавливаемая автоматически `docker-compose.yml`. Вам не нужно добавлять её в ваш файл `.env`, если вы не подключаетесь к внешнему экземпляру MySQL. |

Файл Docker Compose автоматически создаёт базу данных `claraverse` и пользователя `claraverse_user` при первом запуске.

|----------|---------|----------|-------------|

## Ограничение скорости

Все значения — **запросы в минуту**.

| Переменная | По умолчанию | Обязательна | Описание |
|----------|---------|----------|-------------|
| `RATE_LIMIT_GLOBAL_API` | `200` | Нет | Глобальный лимит для всех маршрутов `/api/*`. |
| `RATE_LIMIT_PUBLIC_READ` | `120` | Нет | Публичные конечные точки только для чтения. |
| `RATE_LIMIT_AUTHENTICATED` | `60` | Нет | Запросы аутентифицированных пользователей. |
| `RATE_LIMIT_WEBSOCKET` | `20` | Нет | WebSocket-подключения в минуту. |
| `RATE_LIMIT_IMAGE_PROXY` | `60` | Нет | Запросы прокси изображений (предотвращает злоупотребление пропускной способностью). |

## Контрольный список для продакшена

Перед переходом в продакшен проверьте эти настройки:

```bash
# Обязательные изменения
ENVIRONMENT=production
ENCRYPTION_MASTER_KEY=<ваш-сгенерированный-ключ>
JWT_SECRET=<ваш-сгенерированный-секрет>
MYSQL_ROOT_PASSWORD=<надёжный-пароль>
MYSQL_PASSWORD=<надёжный-пароль>

# Установите на ваш реальный домен
VITE_API_BASE_URL=https://api.yourdomain.com
VITE_WS_URL=wss://api.yourdomain.com
ALLOWED_ORIGINS=https://yourdomain.com
FRONTEND_URL=https://yourdomain.com
BACKEND_URL=https://api.yourdomain.com
```
