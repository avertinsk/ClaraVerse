# Часто задаваемые вопросы

## Какие ИИ-провайдеры поддерживает ClaraVerse?

ClaraVerse поддерживает **OpenAI**, **Anthropic**, **Google AI (Gemini)**, **Ollama** и любую **OpenAI-совместимую конечную точку** (LM Studio, vLLM, Groq, Together AI и т.д.). Вы можете подключить несколько провайдеров одновременно и свободно переключаться между ними.

## Могу ли я использовать ClaraVerse без API-ключа?

Да. Подключите локальный провайдер, такой как **Ollama** или **LM Studio**, и запускайте модели с открытым исходным кодом полностью на вашем собственном оборудовании. API-ключ или подключение к интернету не требуются.

## Каковы системные требования?

| Требование | Минимум |
|------------|---------|
| ОЗУ | 4 ГБ свободно |
| Диск | 10 ГБ свободно |
| Docker | 20.10+ |
| Docker Compose | v2+ |

ClaraVerse запускает шесть контейнеров (фронтенд, бэкенд, MongoDB, MySQL, Redis, SearXNG), поэтому убедитесь, что Docker имеет достаточно выделенной памяти.

## Где хранятся мои переписки?

- **На стороне клиента (основное):** Браузер использует **IndexedDB** в качестве основного хранилища переписок. Все данные чата хранятся локально в браузере по умолчанию.
- **На стороне сервера (опционально):** Если вы включите синхронизацию чата, MongoDB хранит зашифрованные данные синхронизации. Это не включено по умолчанию — пользователь должен выбрать эту опцию.

Все данные остаются на вашей инфраструктуре. Ничего не отправляется внешним сервисам.

## Как сделать резервную копию моих данных?

Сделайте резервную копию Docker-томов, содержащих ваши базы данных:

```bash
# Остановить стек
docker compose down

# Резервная копия данных MongoDB
docker run --rm -v claraverse_mongodb-data:/data -v $(pwd)/backups:/backup \
  alpine tar czf /backup/mongodb-backup.tar.gz -C /data .

# Резервная копия данных MySQL
docker run --rm -v claraverse_mysql-data-new:/data -v $(pwd)/backups:/backup \
  alpine tar czf /backup/mysql-backup.tar.gz -C /data .

# Перезапуск
docker compose up -d
```

:::note
Префикс имени тома зависит от имени директории, куда вы клонировали проект (например, `claraverse_mongodb-data`, если директория называется `ClaraVerse`). Используйте `docker volume ls` для проверки точных имён томов в вашей системе.
:::

:::tip
Автоматизируйте резервное копирование с помощью cron-задачи. Также сделайте резервную копию вашего файла `.env` — он содержит ваш `ENCRYPTION_MASTER_KEY`, который необходим для расшифровки пользовательских данных.
:::

## Как добавить больше пользователей?

Пользователи самостоятельно регистрируются через страницу входа ClaraVerse. **Первый пользователь**, который зарегистрируется, автоматически становится администратором — без кода приглашения или специальной настройки. Все последующие пользователи получают стандартную роль `user`. Администраторы могут продвигать пользователей или корректировать их лимиты из [Панели администратора](./admin/overview_ru.md).

## Мои данные приватны?

Да. ClaraVerse полностью self-hosted. Ваши переписки, файлы и учётные данные остаются на вашей собственной инфраструктуре. Веб-поиск проходит через self-hosted экземпляр SearXNG. Единственные внешние вызовы — к ИИ-провайдерам, которые вы явно настроите.

:::warning
Если вы используете облачные ИИ-провайдеры (OpenAI, Anthropic, Google), ваши запросы и ответы проходят через их API. Используйте локальные модели через Ollama для полностью офлайн, полностью приватной работы.
:::

## Как обновить ClaraVerse?

```bash
git pull origin main
docker compose up --build -d
```

Это загружает последний код, пересобирает контейнеры и перезапускает стек. Ваши тома данных сохраняются при обновлениях.

## Могу ли я использовать обратный прокси?

Да. ClaraVerse работает за **Nginx**, **Caddy**, **Traefik** и другими обратными прокси. Направьте прокси на порт `80` (фронтенд) и убедитесь, что WebSocket-подключения к порту `3001` (бэкенд) правильно перенаправляются.

Минимальный пример Nginx:

```nginx
server {
    listen 443 ssl;
    server_name clara.example.com;

    location / {
        proxy_pass http://localhost:80;
    }

    location /api/ {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## Какие порты использует ClaraVerse?

| Порт | Сервис |
|------|--------|
| 80 | Фронтенд (Nginx) |
| 3001 | API бэкенда + WebSocket |
| 27017 | MongoDB |
| 3306 | MySQL |
| 6379 | Redis |

SearXNG работает во внутренней Docker-сети и по умолчанию не доступен на хосте.

:::tip
В продакшне открывайте только порты **80** (или 443 через обратный прокси) и **3001** для сети. Держите порты баз данных (27017, 3306, 6379) ограниченными localhost или Docker-сетью.
:::
